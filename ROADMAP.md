# ROADMAP.md — Pixel Factory 长期演进规划

> 本文档记录 Pixel Factory 项目的方向性规划、待解决的技术问题、以及各阶段的功能目标。
> 目标：将 Pixel Factory 从"AI 试玩 Demo"演进为**专业级的独立游戏像素资产创作工作流工具**。

---

## 底层核心问题：文本生成像素画的架构局限

当前方案依赖 LLM 通过 RLE 文本协议推演像素网格。这种方式存在以下**系统性限制**，是整个平台需要持续对抗的核心矛盾：

### 1. 维度鸿沟 (Dimensionality Gap)
LLM 将 2D 网格展平为 1D Token 序列，坐标 (x,y) 和 (x,y+1) 在序列中被强行隔开。模型注意力机制无法天然建立 2D 邻域、轮廓闭合、旋转轨迹的直观感知。

### 2. 像素级漂移 (Flickering & Drift)
概率生成模型缺乏刚体物理或骨骼绑定约束。角色旋转、转身等动作时，易出现孤立噪点、边缘抖动、结构比例变形。

### 3. 长度约束幻觉
16x16 (256 像素) 勉强精确；32x32 (1024 像素) 或更高时，RLE 输出极易多/少像素导致整图位移错位。当前解码器通过 padding/truncation 兜底，但位移后的图像已不可用。

### 4. 调色板索引乱序
模型推演下一帧时可能混淆调色板索引（如把轮廓线和阴影的索引互换），引发色块混乱。

## Phase 1: 体验加固 & 生成链路增强（近期）

### 1.1 AI 引导词模板库 (Prompt Presets)
专门针对像素画物理运动的提示词模板。例如：
```
[转身运动]: 保持像素总量，将边缘轮廓向 Y 轴中线挤压，同时将面部特征向旋转方向平移，边缘像素做阶梯级渐变处理。
```
- 在 AIGenerator 组件中加入模板下拉选择
- 覆盖常见运动：转身、跳跃、攻击、跑动循环、呼吸/idle

### 1.2 孤立像素自动降噪 (Isolated Pixel Healer)
- 前端加入一键检测并擦除孤立浮空像素的滤镜
- 基于连通域分析（4-connected），面积小于阈值的像素块自动标记为噪点
- 这是 LLM 生成时最常见的瑕疵类型，修复后可大幅提升观感

### 1.3 2D 文本锚点表达
- 向 LLM 传递带行号的 2D 字符矩阵（非 1D 展平数组）
- 利用模型处理多行代码/排版对齐的能力增强空间位置感知
- 服务器端在 system prompt 中改用 `[行0: ...] [行1: ...]` 格式

### 1.4 逐行 RLE 兜底增强
- 当前 decodeRLE 已经支持按行数组格式解码（animation 端已使用）
- `/api/generate` 端点也应当支持返回行数组格式，减少单次 RLE 长度误差概率

## Phase 2: 智能辅助 & 半自动创作（中期）

### 2.1 关键帧插帧引擎 (Smart Tweening)
- 用户手工绘制首帧 (Frame 1) 和尾帧 (Frame N)
- AI 负责计算并生成中间过渡帧 (In-betweens)
- 首尾锚点大幅提升插帧的结构准确度

### 2.2 局部 AI 刷子 (Selective AI Paintbrush)
- 用户框选锁定部分区域（如"蘑菇头部不动"）
- AI 只影响未锁定区域（如"下半身长出腿跑动"）
- 降低整体生成导致的画面崩塌概率
- 实现方式：mask + 局部 RLE 替换

### 2.3 多模态视觉反馈闭环 (Vision Feedback Loop)
当前链路：纯文本 → LLM → RLE → 解码 → 渲染
目标链路：纯文本 → LLM → RLE → 解码 → **渲染为图片 → 多模态模型（Gemini Vision）评估 → 自动校正** → 渲染
- 多模态模型能发现"后脑勺缩水了"、"转身时眼睛没对齐"等问题
- 自动给出微调指令进行第二轮修正

### 2.4 骨骼 + 蒙皮分层生成 (Layered Generation)
- 第一轮：生成单色轮廓线骨架 (Silhouette)，确保运动轨迹和轮廓转面无误
- 第二轮：对骨架内部进行调色板填充和光影渲染 (Shading)
- 无需更复杂的模型调用——通过多轮 system prompt 约束即可实现

## Phase 3: 专业级独立游戏资产工作流（远期）

### 3.1 Aseprite 格式互通
- 支持导入 `.ase` / `.aseprite` 专业像素软件格式
- 导出为 Aseprite 格式
- 支持切片精灵图 (Spritesheet Sprite Slicing)

### 3.2 调色板自动提取与映射 (Auto Palette Harmonizer)
- 上传真实照片 → AI 自动提取极简 8-bit 调色板
- 将照片转换为 16x16/32x32 像素图标
- 作为动画的初始状态输入

### 3.3 游戏引擎导出深度集成
- 一键生成 Unity、Godot、Phaser 适配的 Spritesheet + JSON 配置文件
- 当前已支持 Unity/Phaser 帧坐标 JSON，需扩展更多引擎格式
- 支持 CSS 动画代码导出

### 3.4 云端项目同步
- 当前使用 localStorage 持久化
- 远期支持云端存档、项目多设备同步
- 可选：社区作品分享画廊

## 技术债务 & 架构改进

### 前端
- **状态管理**：当前所有状态集中在 `App.tsx`，props drilling 严重（6-7 层）。中期可引入 `useContext` 或 zustand。
- **动画帧同步**：`useEffect` 中通过 `JSON.stringify` 深度比较同步帧，16x16 尚可，64x64 有性能隐患。
- **GridCanvas 渲染性能**：当前使用 CSS Grid + 数百个 `<div>` 单元格，大尺寸（64x64 => 4096 个 div）渲染成本高。可考虑迁移到 `<canvas>` 渲染。
- **i18n**：大量 UI 标签硬编码为中文，抽取为 locale 文件以支持国际化。

### 后端
- **AI 调用容错**：当前对 Gemini API 失败没有重试逻辑，结构化 JSON 解析失败会直接 500。
- **请求超时**：无显式 timeout 控制，大尺寸/多帧生成可能挂起。
- **模型选择**：硬编码 `gemini-3.5-flash`，应可配置。
