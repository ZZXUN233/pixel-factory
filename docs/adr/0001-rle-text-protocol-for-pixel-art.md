# 0001 — 使用 LLM + RLE 文本协议生成像素画

当前版本通过 Gemini 3.5 Flash 生成 RLE 文本编码的像素网格，前端解码后渲染。这是一个"文本空间推理图像"的架构选择。

## 背景

Pixel Factory 最初是 Google AI Studio 上构建的应用，运行环境对 Node.js 依赖友好，但对图像处理模型（如 Stable Diffusion、Imagen）的调用来之不易。选择纯文本协议 RLE 使得唯一需要的 API 能力就是文本生成，无需多模态视觉 API。

## 考虑的替代方案

- **端到端视觉模型（Imagen / Stable Diffusion）**：生成质量最高，但需要额外的图像生成 API，且生成结果是光栅图，无法直接得到像素网格级编辑能力。
- **ComfyUI / ControlNet + Pixel Art 微调**：控制力强，但部署成本高，与 AI Studio 无服务器架构冲突。
- **让 LLM 直接输出 JSON 像素数组**：等价于 RLE 但更冗长，在 Token 受限场景下不可行。

## 影响

- **优势**：极低延迟，无需多模态 API，RLE 本身是像素画的自然压缩格式，可直接预览和编辑。
- **限制**：LLM 缺乏 2D 空间连续性感知，高分辨率（>32x32）下计数误差频发，生成结果有像素漂移和结构变形问题。这些限制在 ROADMAP.md 中有详细记录和缓解方案。

## 后续演进方向

随着平台脱离 AI Studio 环境，可逐步引入多模态视觉反馈闭环（Phase 2.3）和局部 AI 刷子（Phase 2.2），最终过渡到"LLM 推理轮廓 + 视觉模型精修"的混合架构。但 RLE 作为传输协议仍会保留，因为它提供了 AI Studio 闭包之外的工具链中罕见的"纯文本可读、可编辑、可验证"特性。
